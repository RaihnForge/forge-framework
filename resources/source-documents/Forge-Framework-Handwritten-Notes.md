## Page 1 — FORGE Cycle Overview with Wisdom Principles

The FORGE cycle is laid out top-to-bottom with each phase's mechanical function in the center column and a paired **wisdom principle** in the right column.

### Cycle Flow

| Phase | Function | Mechanics | Wisdom Principle |
|-------|----------|-----------|------------------|
| **F — Feed** | Input | Proaction (Concept) / Reaction (Event) | "Failure: Key to Success" |
| **O — Organize** | Plan / Action | Scope, Time, Cost | "Vision + Mission + Values + Market + Industry" |
| **R — React** | Execution Process | Monitor + Control | "Leadership (Admin / Manage)" |
| **G — Generate** | Output | Deliverables | "Walk Softly + Carry a Big Stick / Know Them by Fruits of Labor" |
| **E — Express** | Evidence / External Communication / Export | — | "Who are you? What do you do? How do you do?" |

### Wisdom Principles Explained

**Feed — "Failure: Key to Success"**
There will be many inputs, and they will likely represent corrective measures more often than not — failures that serve as stepping stones to success.

**Organize — "Vision + Mission + Values + Market + Industry"**
When organizing input or considering project constraints, everything must be tempered by organizational values, ethics, and mission for prioritization of value and effort.

**React — "Leadership (Admin / Manage)"**
Derived from project management philosophy. Once the action item is in progress, your job is to monitor and control through leadership principles and sound administrative management.

**Generate — "Walk Softly Carry a Big Stick / Know Them by Fruits of Labor"**
Concepts to bear in mind when dealing with output. Be firm and fair with all parties and reasonable with scope. "Know them by fruits" is a call to action — everything we do reflects our character. Be mindful of who you are building yourself to be. (Teddy Roosevelt + biblical reference.)

**Express — "Who are you? What do you do? How do you do?"**
Beyond the output/deliverable there are extra tangibles:
- **Evidence** — compliance standpoint: know the results, state of trackers at completion.
- **External Communication** — to co-workers, collaborators, like-minded people, the public, dissenters: what happened, why it matters, why it should be celebrated, what can be learned.
- **Export** — ensure tangible relevant after-action effects. Learn from every experience. (Closely parallels project closing concepts in PM.)

## Page 2 — Detailed Breakdown of Each Phase's Mechanics

*(Labeled "3 of 3" on the original)*

### Feed (Input)
Input types: **Strategy**, **Event**, **Situation**

### Organize (Action)
- **Action Plan (Meeting)** — structured with Header (Title, Date, Attendees), Agenda (A, B, C), Discussion
- **Action Ticket** — captures: Action Item #, Who, What, When, Where, How
- **Org Ethics** — Mission, Vision, Values + People
- **Action Items** — numbered list (Action Item 1, 2, 3...)

### React (Execution Process)
- **Action Log** — numbered entries (001–005...)
- **Monitor** — Follow-Up Meetings, Trackers, Reports
- **Control** — Leadership, Policy & Process, Administration

### Generate (Output)
- **Deliverables** → Outcome → Product → Result

### Express
- Evidence
- External Communication
- Export
- Subcategories: Outgoing, Compliance, Reports, Findings, Product

## Page 3 — Operational Structure / Folders as State Machine

*(Labeled "2 of 3" on the original)*

### Operation — 4S Drive

**Branch of Business (Department)** contains Functional Folders:
- **Projects** — Complex Actions / Temp Processes
- **Binder** — Express: Export + External Communication
- **Archive** — Express: Evidence
- **Actions** — Feed + Organize

### Generate — Process
- Process in Play → Execution + Output of Operational Branch

### React — Monitor + Control Process
- Process document
- Tracker – Log
- Tracker – Log

### State ↔ FORGE Mapping

| State | FORGE Phase |
|-------|-------------|
| Input | Feed |
| Action | Organize |
| Execute | React |
| Output | Generate |
