# The Forge Framework

A theory of organizational management for individuals and organizations.

**Creator:** Joshua Keyes

> **Naming:** The framework is called "Forge Framework" — never "Forge Foundation." The directory name `ForgeFoundation` is just the repo folder.

## Three-Tier Doc Strategy

1. `docs/` — Master Reference (source of truth, extended outline)
2. `evidence/` — Stories, proofs, studies, anecdotes (feeds the guide)
3. `guide/` — Published guide (WIP, draws from docs + evidence)

## Key Framework Conventions

- **Miller's Law:** 7 ± 2 (range 5–9) as a conscious design constraint for departments, categories, and groupings.
- **FORGE Cycle:** Feed → Organize → React → Generate → Express (each step has a wisdom principle).
- **Personal Forges (Maslow-mapped, 5 forges):** HomeForge, HealthForge, UnchosenForge, RaihnForge, PaladinForge.
- **Business template:** Admin, Clientele, Product, Resources.
- **Functional Folders (state machine):** Action Log, Queue, Projects, Binder, Archive. Folders represent operational state, not just storage.
- **Three doc types:** Monitor (MON), Control (CON), Evidence. Outdated MON/CON becomes Evidence.
- **Void Forge:** Advanced concept for pre-discovery ambiguity, penciled-in potentials, and do-not-do tracking.
- **Projects END, Processes are CONTINUOUS.**
- **Four C's gate:** Comfort → Confidence → Competence → Compliance. Over-ambition before clearing these stages is a primary failure mode.
- **Departments are flexible** — not a fixed list. Aim for 5–9 per Miller's Law.

## Writing Guidelines

- **Accuracy first.** Verify all factual assertions — especially statistics, studies, and attributed quotes.
- **Voice:** Direct, practical, philosophical. No fluff, no filler.
- **Allegories are core:** Plot of Land / Fiefdom (Lord = Control, Scribe = Monitor, Castle = Folders, Village = Processes, Output = Evidence).
- **Universal design principles:** Scalability, maintainability, reliability. Methodology-agnostic infrastructure.
