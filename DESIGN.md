# Forge Framework — Design Vision

> **Document Created:** 2026-04-03
> **Status:** Initial

## Vision

A management framework that scales from one person to an organization, grounded in universal design principles. The FORGE Cycle (Feed → Organize → React → Generate → Express) is the engine. Personal Forges (5 life domains mapped to Maslow) provide the structure. Miller's Law (7±2) sets the constraint.

This is methodology, not software. The product is a written framework — docs, evidence, guides.

## Design Principles

1. **Allegory-driven** — Plot of Land, Fiefdom, Scribe/Lord/Castle metaphors make abstract principles concrete and memorable.
2. **Universal applicability** — Works for individuals (Personal Forges) and organizations (departments, projects). Same cycle, different scale.
3. **Three document types** — Monitor (MON, operational state), Control (CON, rules/architecture), Evidence (EVD, cases/anecdotes). Every artifact fits one category.
4. **Projects have endpoints; processes are continuous** — The distinction matters. Projects complete. Processes endure.
5. **Decision gates** — Comfort → Confidence → Competence → Compliance. Progressive commitment, not premature optimization.

## UX Philosophy

The "user" is someone adopting the framework. The UX is:
- **Docs/** — Master reference material. Read to understand the theory.
- **Evidence/** — Stories and case studies. Read to see the theory applied.
- **Guide/** — Published methodology. Read to implement it.

Modular, browsable, grounded. Not fluffy theory — practical philosophy with systems thinking.

## Aesthetic Direction

- Written framework only — no visual UI
- Allegory and metaphor as comprehension tools
- Structured markdown with clear hierarchy
- CON/MON/EVD classification visible in document headers
- Practical tone — grounded, actionable, not academic

## What Success Feels Like

A team adopts the FORGE Cycle. They organize work into functional folders (Action Log, Queue, Projects, Binder, Archive). Decisions move through gates (Comfort → Confidence → Competence → Compliance). Departments stay within 7±2 scope. Documentation is classified (MON/CON/EVD) and maintained. The framework disappears into how they work — it's not overhead, it's just how things are organized.
